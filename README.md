# discr-math-18-team
найкоротший шлях на прямокутній ділянці
ЗВІТ
    Проєкт: Пошук найкоротшого шляху на прямокутній ділянці
1. Мета роботи
    Розробити програму, яка за заданою матрицею висот визначає найкоротший шлях між двома точками на прямокутній ділянці, враховуючи зміну висоти. Додатково —          візуалізувати отриманий маршрут на графічній діаграмі.
2. Постановка задачі
    Вхідні дані:
    •	CSV-файл із матрицею висот.
    •	Стартова точка: координати (рядок, стовпець).
    •	Кінцева точка.
    •	Крок між сусідніми точками (за замовчуванням — 1).
    Потрібно:
    1.	Прочитати матрицю висот.
    2.	Знайти найкоротший шлях між двома точками з урахуванням перепаду висот.
    3.	Побудувати графічну візуалізацію висотної мапи та знайденого шляху.
    Для пошуку використовується алгоритм Дейкстри, який дозволяє працювати з вагами — у цьому випадку вага переходу залежить від різниці висот.
3. Опис програми
    Основні частини програми:
    1.	arguments_pars()
      Обробка аргументів командного рядка (шлях до файлу, координати точок, крок).
    2.	read_matrix(path)
      Зчитування матриці висот із CSV-файлу.
    3.	coords_pars(coord)
      Перетворення координат з формату "рядок,стовпець" у кортеж (row, col).
    4.	dijkstra(matrix, start, end, step)
      Реалізація алгоритму Дейкстри для пошуку найкоротшої траєкторії.
    5.	visualize(matrix, path)
      Графічний вивід висотної карти та знайденого шляху.
4.	main()
    Основна функція, яка об'єднує всі частини.
    Алгоритм роботи програми
    1.	Отримання параметрів.
    2.	Зчитування висот у двовимірний список.
    3.	Ініціалізація таблиці дистанцій (заповнена ∞).
    4.	Запуск алгоритму Дейкстри:
      o	пріоритетна черга (heapq),
      o	чотири напрямки руху,
      o	оновлення мінімальних відстаней,
      o	збереження попередників для відновлення повного шляху.
    5.	Після досягнення кінцевої точки — відновлення маршруту.
    6.	Виведення результатів:
      o	довжина шляху,
      o	список координат,
      o	візуалізація.
5. Результати роботи
    Програма виводить:
    1. Відстань:
      Числове значення довжини найкоротшого шляху.
    2. Шлях:
      Послідовність координат від старту до фінішу.
    3. Графічне зображення:
      •	Висотна карта (terrain).
      •	Червона лінія — знайдений найкоротший шлях.
    6. Висновки
    У ході виконання роботи було:
      •	Реалізовано алгоритм пошуку найкоротшого шляху на нерівній поверхні.
      •	Сформовано структуру даних для роботи з матрицею висот.
      •	Використано алгоритм Дейкстри з вагами, що враховують різницю висот.
      •	Здійснено графічну візуалізацію результату.
    Програма може бути застосована в задачах:
      •	моделювання пересування на картах,
      •	геоінформаційних систем (GIS),
      •	планування маршрутів на рельєфних поверхнях.
